Installation

Drop the extension into pyRevit and reload.

The suite is a standard pyRevit extension. Clone or copy IcZScripts.extension into your pyRevit extensions folder, then reload — the SuiteTools tab appears automatically.

shell
# 1. Clone into a folder pyRevit watches
git clone https://github.com/MrGezz/IcZScripts.extension.git

# 2. Register the parent folder in pyRevit
#    Settings → Custom Extension Directories → add the folder

# 3. Reload pyRevit
pyrevit reload
  • pyRevit 6.4+ on the IronPython 2.7 engine (engine files must be CRLF).
  • Revit 2023 – 2026. The eid() shim handles the ElementId Int32 → Int64 change automatically.
  • No per-bundle setup. pyRevit adds lib/ to sys.path for every tool, so from icz import … just works.

SPEC / CMMN Automation AA GmBH.panel

Full-lifecycle parameter management for MEP elements — phases 0 through 8.

AA is the flagship coordination tool. It walks the model through a numbered pipeline: it first repairs connectivity, then de-duplicates geometry, writes standardized SPEC/CMMN parameters, assigns IFC export types, and finally routes elements to the correct worksets. A pre-scan review grid shows every proposed change before anything is committed.

  • Phase 0 — MEP Connection Fixer. Scans open-ended ducts, pipes and trays, seals unsealed end-caps via subtransactions so they don't show up as false duplicates downstream.
  • Phase 1a/1b/1c — Duplicate & short-duct cleanup. Flags parallel curves of the same category/size sharing a centerline offset < 1 mm with matching connectors; merges duct segments under 50 mm.
  • Phase 4 — CMMN standard parameters. Writes SPEC_SIZE, SPEC_SYSTEM, SPEC_QUANTITY and SPEC_ARTICLE_EN/DE from geometry + lookup tables, and verifies clamp counts on ChemicalBox / corrosive conduits.
  • Phase 7 — IFC assignment. Maps Revit categories to custom IFC export types (e.g. generic models → IfcFlowFittingType / IfcDuctAccessoryType).
  • Phase 8 — Workset orchestration. Routes curves to designated worksets (DX_STB, DX_EXH, DX_ELT, DX_PG, DX_RR).
Review-first UX. A pre-scan grid lists every proposed change. Click any row for single-item Temporary Isolate (ActiveView.IsolateElementsTemporary()) and click-to-zoom navigation. The post-run report renders live metric chips — successful writes, skips, workshared lockouts — instead of a TaskDialog.
API: FilteredElementCollector API: SubTransaction View: IsolateElementsTemporary Data: spec_mappings.json

MEP Workbench FIT GmBH.panel

A four-page modeless workbench for routine MEP cleanup, routing & parameter maintenance.

FIT consolidates four formerly-standalone scripts (MEPPairing, copy_param, DX_B37_X_and_Y, BIM_MI_Script_002) into one navigated window. Each page is an independent task with its own review grid and isolate-on-click.

Fitting Audit

Three rule cards (A/B/C) flag fittings & accessories that violate configurable rules — missing IFLS, mismatched system, orphaned components. Rule B uses an O(n) proximity search within a 1500 mm bounding box; whitelists come from fitting_config.json.

Coordinates

Writes world X/Y of each selected element into two text params (default H.2 Position X / H.3 Position Y) in metres to 3 decimals, after an editable global offset. Overwrite-guard leaves existing values untouched when off.

Pipe / Duct → Flex

Converts rigid runs to flex via FlexPipe.Create / FlexDuct.Create. Replace swaps per element; Containment walks the centerline through fittings and lays a parallel flex, keeping originals. Vertex count, sag & Chaikin smoothing tunable.

IFLS Param Sync

Reads IFLS / AKS Infineon / Prozessmedium from source fittings in linked files and writes them onto host targets matched by MEP system type name, gated by a cylindrical proximity bound.

Project Setup Utility PSU GmBH.panel

One dashboard consolidating link worksets, project info & level reassignment.

PSU supersedes the former standalone LNK_, Level and project-info scripts, merging them into a single modeless window with three pages.

Link Workset Assignment

Batch-sorts Revit / CAD / NWD link instances into worksets matching _LINKS_REVIT, _LINKS_CAD, _LINKS_NWD by filename pattern — instance and parent type assigned inside one Transaction.

Project Info

Reads/writes only standard built-in ProjectInfo params (Organization, Author, Status…). Non-built-in params (MagiCAD, IFC GUIDs) are filtered out to protect document health. An IFD reminder banner is driven by psu_config.json.

Levels

Bulk-reassigns reference level across categories while preserving physical elevation, computing a delta offset so elements never move. Hosted instances are safely skipped.

Security Vault

Password-gated verification calling DxMY.dll → SecurityVault.KeyManager via clr.AddReferenceToFileAndPath to keep workflows from soft-locking on license windows.

Levels — elevational offset compensation
ΔZ = Z_old − Z_new Offset_new = Offset_old + ΔZ // written to INSTANCE_FREE_HOST_OFFSET_PARAM
API: ProjectInfo API: Level / WorksetTable API: ReferenceLevel Config: psu_config.json

Model Cleanup CD Tools.panel

Two-tool model-hygiene suite — Ghost Purge + Connection Fix. All results in-window.

Ghost Purge

Finds overlapping duplicate elements, keeps the connected copy, deletes the floating ghost. Groups by (CategoryId, TypeId) first — O(n) per group, not O(n²). Connected element wins; ties break to the smaller ElementId. SuperComponent guard never drops nested members.

Connection Fix

Seals open MEP connector pairs within 20 mm via spatial-grid bucketing. Re-fetches connectors at connection time, checks domain compatibility (Piping / HVAC / CableTrayConduit), and isolates each attempt in its own SubTransaction so one failure never rolls back the run.

Stale-data exclusion. Both scan caches are keyed to model state. Any external DocumentChanged event marks the caches dirty, so reopening either tool prompts a fresh Rescan rather than acting on stale rows.
API: SubTransaction API: GetTransform / BoundingBox Algo: spatial grid bucketing Event: DocumentChanged invalidation

BOQ Lookup RO1 Parameter.panel

Classifies MEP elements by material / fitting segment & nominal width, then resolves BOQ codes.

RO1 parses element sizes — including messy inch strings and fractions — into metric Nominal Widths (NW), then cross-references a master index from IFT_BOQ.xlsx to write SPEC_DESIGNATION and DESCRIPTION.

boq_engine.py — automated size resolution
def lookup_boq(element, doc, read_param_func):
    size_str = read_param_func(element, "SPEC_SIZE", doc)

    # Fallback to Revit native size if SPEC_SIZE is empty
    if not size_str:
        p = element.get_Parameter(DB.BuiltInParameter.RBS_CALCULATED_SIZE)
        if p:
            size_str = p.AsString()

    # Parse complex inch strings into metric Nominal Widths (NW)
    if size_str:
        sub_parts = re.split(r'[-x×](?!\d/)', size_str)
        all_nws = []
        for sp in sub_parts:
            if '"' in sp:                       # inch + fraction (e.g. 1 1/2")
                clean = sp.replace('"', '').strip()
                total = sum(float(f.split('/')[0]) / float(f.split('/')[1])
                            if '/' in f else float(f) for f in clean.split())
                all_nws.append(int(INCH_TO_NW.get(total) or round(total * 25.4)))
            else:                               # already metric
                found = re.findall(r'\d+\.?\d*', sp.replace(",", "."))
                if found: all_nws.append(int(round(float(found[0]))))

        if all_nws:
            max_nw = str(max(all_nws))
            res = boq_data.BOQ_SIZE_INDEX.get((mat_seg, fit_seg, max_nw))
            if res: return res

    return (None, "Size '{}' not found in BOQ Index".format(size_str))
API: BuiltInParameter.RBS_CALCULATED_SIZE Parse: regex inch/fraction → NW Data: IFT_BOQ.xlsx index

Shared Library lib/icz/

One source of truth per concern — imported by every tool, duplicated nowhere.
  • icz.theme — single source for Light/Dark palettes. Tools call theme.apply_theme(self, "dark"|"light"); controls bind to {DynamicResource <token>} and repaint automatically.
  • icz.review_grid — shared in-window grid builder producing [CheckBox | ElementId | Family & Type (+ badge)] rows with hover, click-to-navigate and status pills. AA and CD both consume it.
  • icz.revit_compat — the eid() shim returns ElementId.Value on 2024+ (Int64) and ElementId.IntegerValue on 2023 (Int32). Import everywhere an ElementId becomes a key.
  • icz.modeless — ExternalCall wraps Revit's IExternalEventHandler / ExternalEvent so modeless windows raise model-mutating work into a valid API context without hand-managing the event plumbing.
usage across every bundle
from icz import theme
from icz.revit_compat import eid
from icz.modeless import ExternalCall

class ToolWindow(Window):
    def __init__(self):
        theme.apply_theme(self, "dark")            # one call, whole window themed
        self._mutate = ExternalCall(self._commit)  # safe API context for edits
        self._navigate = ExternalCall(self._zoom)  # separate event → never races

    def key_for(self, element):
        return eid(element.Id)                      # Int32/Int64 handled by shim

Conventions & Design Principles

The rules that keep the suite coherent and crash-free.

IronPython 2.7 constraints

  • No f-strings — use .format(). No open(encoding=…) — use io.open() / codecs.open().
  • TemplateBinding padding: Padding is valid only on Border / Control subclasses — never on Grid, StackPanel, DockPanel, WrapPanel. Use Margin on children or wrap in a Border.
  • Glyphs: a Segoe MDL2 Assets glyph must sit in its own glyph-only TextBlock, declared before the label's TextBlock — never mixed in one block, or the glyph renders as a square.
  • Revit 2024+: always use the eid() shim; never call .IntegerValue directly. Engine files must be CRLF.

Design principles

  • Single source of truth per concern — theme, grid builder, ElementId compat and each mapping table live in exactly one file.
  • No auto-scan on open. Every window opens instantly; grids fill only on explicit Rescan, Dry Run, or scope change.
  • In-window results. No pyRevit output window, no end-of-run TaskDialog — results render as metric chips and click-to-zoom tables inside the tool.
  • Modeless, persistent engine. __persistentengine__ = True keeps ExternalEvents alive; navigation and mutation route through separate ExternalCall instances so they never race.
Theming standard. AA is the reference theme — patch other tools to match by function, not 1:1 pixels. The suite standardizes on one rounded-corner scale (window shell 6, title bar 6,6,0,0, cards 6, buttons 4–6, inputs 3–4, scroll thumb 4, traffic-light dots 7). ComboBoxes stay themed via DynamicResource tokens.
