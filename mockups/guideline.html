Section 3 · Model Structure & Worksets — IcZ.PdfView (PDFium render) · annotations shown

3.1 Workset partitioning

Model elements shall be assigned to worksets by discipline and system function, following the schedule in Appendix B. Link instances occupy dedicated ZZ_LNK_* worksets and are never left on a working set.

3.2 Level of Information (LOI)

Every deliverable model is scored against the firm's Model Checker LOI ruleset before issue. Elements failing mandatory parameters are corrected — no waiver is granted for SPEC_* fields.

  • SPEC_SYSTEM, SPEC_SIZE and SPEC_QUANTITY written by the QAQC pipeline
  • Openings coordinated through the MEP Opening workflow, one marker per crossing
  • Exported IFC typing driven by the classification table, not by hand

3.3 Issue procedure

Weekly issues run through the transmittal tool with the project profile; the sheet index accompanies every package.
